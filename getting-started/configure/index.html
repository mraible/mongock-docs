    If you have a local copy of your site installed you can edit this file directly, or you can edit and commit the file using the Github UI. The default site.json file looks like so:

    Basic settings

    When you make changes to this file and deploy them, your site will automically update to reflect your new settings:

    • name ‚Äì this is the name or title of your site as it appears in the header
    • subtitle (optional) ‚Äì this is the short subtitle that appears directly below your site name
    • description ‚Äì this is used as your default metadata page description, and can be overridden on a page-by-page basis as needed
    • footer (optional) ‚Äì this is the plain text that appears in the bottom footer
    • githubUrl ‚Äì this is the address to your Github repository, used to generate edit links
    • githubBranch ‚Äì this is the main of your primary Github branch (set to ‚Äúmain‚Äù by default)
    • navigationStyle ‚Äì you can set your navigation to horizontal (default), or vertical if you prefer a top navigation bar
    • emoji (optional) ‚Äì this is the emoji used for your favicon
    • url ‚Äì this is the main URL to your site on Netlify (this is not currently used but seems wise to track in config)

    Enable or disable features

    • enableSearch ‚Äì this will enable or disable the Elasticlunr search engine
    • enableContact ‚Äì enable or disable the Netlify contact form
    • enableDarkMode ‚Äì enable or disable darkmode functionality
    • enableEditButton ‚Äì enable or disable the ‚Äúedit on Github button‚Äù
    • enableDatestamp ‚Äì enable or disable the date stamp that appears on the top of each page
    • enableGithubLink ‚Äì enable or disable the Github link and icon that appears in the footer
    • enableNetlifyCMS ‚Äì enable or disable Netlify CMS (false by default)
    • enableEncryption ‚Äì enable the logout button if you are using encryption (false by default)
    • enablePageNavigation ‚Äì enable or disable the built-in navigation that appears on the bottom of each page

    Did you know? If you enable the optional Netlify CMS integration, all of these settings can also be changed via the CMS.

    Basic Netlify configuration

    There are probably also a few settings you may wish to change on Netlify:

    • You can change your site name and URL to something more personal under General > Site settings > Site details > Site name
    • If you leave your contact form enabled, you may wish to set up notifications under Forms > Form notifications
    • You may wish to change your deploy settings under Build & deploy > deploy contexts
    • You may wish to explore other Netlify settings!
